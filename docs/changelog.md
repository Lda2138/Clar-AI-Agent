# 📖 Clar AI 助教项目开发日志 & 版本记录

本文档用以记录《随机信号分析》AI 助教 Clar 项目各版本的开发日志与更新履历。

## [v6.0.1] - 2026-06-12
### 实时流式蒙特卡洛动画引擎与极致后端瘦身
- **解决蒙特卡洛 O(N^2) 性能灾难**：
  - 彻底移除了对 150,000 点超长拼接数组进行无意义的自相关和频域分析（规避了高达 225 亿次的无用计算），改用极速的 O(N) 线性时间直方图算法生成 PDF。计算耗时从 10+ 秒骤降至 5 毫秒内。
- **重构流式渐进生长动画 (Progressive Rendering)**：
  - 废弃后端一次性吐出并全量绘制的方式。采用 JS 原生定时器逐根叠绘波形，并在前端毫秒级实时计算并渲染概率密度直方图的生长与收敛过程。实现了媲美真实硬件示波器余晖的“动态交织”绝佳视觉体验。
  - 修复了 JS 模板字符串导致图表无法加载的语法错漏。

## [v5.0.0] - 2026-06-12
### 全栈深层解耦与代码剥离
- **前端 CSS 彻底剥离**：
  - 终结了 `index.html` 高达 1400 行的臃肿历史，将 300 多行杂乱的内联 `<style>` 完美抽离至统一规范的 `frontend/css/style.css` 目录，实现了样式与 DOM 的绝对分离。
- **后端核心算法引擎微服务化拆分**：
  - 解剖长达 450 行、逻辑混乱的 `core/signal_engine.py`。
  - 将各个领域的算法依物理意义严格拆离至 `core/signal/` 目录下：
    - `generators.py` (生成域)
    - `analyzers.py` (分析域)
    - `filters.py` (滤波域)
  - 保留 `signal_engine.py` 作为门面模式 (Facade) 网关路由器，确保上层所有接口与原有服务零成本双向兼容。

## [v2.1.0] - 2026-06-02
### 雷达追踪与动态图表感知修复
- **修复 Plotly 动态重绘导致的小球死循环（来回跑）问题**：
  - 在“雷达追踪”等多帧动态渲染图表中，Plotly 引擎会高频销毁并重建 SVG 元素。这导致底层的 DOM 不断触发 `mouseout` 和 `mouseover` 事件，使小球不断在鼠标和右下角之间来回飞行（陷入飞出-重置死循环）。现在增加了元素防抖和鼠标位置校验逻辑，即使底层 SVG 元素被销毁重绘，只要鼠标依然停留在感知卡片范围内，小球就会稳稳锁定在鼠标旁。
- **修复悬浮球点击无响应 Bug**：
  - 修复了当小球飞至鼠标下方覆盖原有元素时，意外触发 `mouseout` 导致后台缓存的感知数据被误删，进而引发单击失效的问题。现在小球完美充当“事件透明代理”，悬停点击一气呵成。

## [v2.0.9] - 2026-06-02
### Clar-ball 面板显示冲突修复
- **修复悬浮模式下侧边栏隐藏 Bug**：
  - 修复了在 Clar-ball 模式下，单击小球后 AI 聊天面板无法正确弹出的问题。此前因为面板自带的 `collapsed-clar-ball` （用于在极简模式隐藏面板）优先级最高，导致代码中简单的 `add('expanded')` 被覆盖失效。部分用户因此不得不双击切回 Classic 模式才能看聊天内容，导致了“小球跑到右上角”的错觉。现在单击小球时会主动移除 `collapsed-clar-ball` 样式限制，让侧边栏完美展开并配合鼠标旁的小球协同工作；双击关闭时会将其再次隐藏。

## [v2.0.8] - 2026-06-02
### Clar-ball 运动轨迹与动画平滑过渡修复
- **修复瞬间位移（传送）Bug**：
  - 在 `telemetry.js` 的 `triggerSensing` 逻辑中，当小球首次追踪鼠标时，它会基于原始的右下角 `top: calc()` 和 `right: 32px` 样式直接跳跃。本次更新通过在赋新值前，显式调用 `getBoundingClientRect()` 并使用强制重绘（`void toggle.offsetWidth`），预先填入当前的精准坐标，让 CSS `transition` 可以平滑计算出两点之间的轨迹，实现从右下角向鼠标“飘去”的平滑动画效果。
  - 在 `clearSensing` 中增加了返回过程的动画约束，小球会先飞回初始锚定点（通过计算视窗宽度推演），等待 $0.8$ 秒飞行落地后，再剥离内联坐标样式将控制权平滑交还给原生 CSS 类，从而实现了优雅的“从哪来，回哪去”闭环体验。

## [v2.0.7] - 2026-06-02
### Clar-ball 交互体验修复与版本更新
- **小球悬停归位异常修复**：

## [v2.0.3] - 2026-05-30
### 相位谱图线降噪优化与下采样数据对齐
- **移除相位谱错误下采样**：
  - 后端移除了对 `phase_clean` 和 `phase_noisy` 数组的独立 `_downsample` 下采样，统一与频率 `freqs` 和幅度谱 `spec` 的全尺寸数组对齐，彻底解决了因长度不匹配导致的相位物理频率映射错乱问题。
- **引入幅度阈值相位过滤**：
  - 针对含噪及纯净相位谱，基于信号幅度最大值进行 5% 阈值过滤，仅在主能量频段（信噪比高的频段）保留相位信息并进行线条绘制，避免无谓的零幅背景噪声无序填充相位图。
- **信号与噪声背景分离样式设计**：
  - 提取高能成分为“观测相位谱-信号”图层，使用红色折线+圆点清晰标明。
  - 提取低能噪声背景为“观测相位谱-噪声”图层，使用极低不透明度（`opacity: 0.2`）的淡红散点平铺作为底色，既展示了白噪声随机相位的统计分布特性，又确保了主信号相位的挺拔可读，画面干净高级。
  - 针对纯噪声信号（如高斯白噪声），自适应淡化为全画幅低度透明散点图，完美契合物理事实。
