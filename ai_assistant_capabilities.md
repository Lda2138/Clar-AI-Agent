# 《随机信号分析》AI 助教 Clar 功能与能力指南

本指南详细介绍了本系统中 AI 助教 **Clar** 的核心设计理念、全套功能场景、深度自动化闭环机制以及所配备的底层专业知识库。

---

## 1. 智能体概述

**Clar** 是专门针对《随机信号分析》课程开发的交互式教学与仿真分析智能助手。系统遵循 **"感知 (Perception) — 决策 (Decision) — 控制 (Control)"** 的闭环控制流：
*   **感知**：Clar 实时读取用户在前端设置的信号参数（类型、频率、采样率、信噪比、带宽、扫频上限）以及系统的滤波、卡尔曼仿真结果和统计特征。
*   **决策**：通过自然语言处理，结合专属的多维联动随机信号知识库，精确定位用户所学的核心概念、公式定义与易错误区。
*   **控制**：AI 可以**直接指挥前端页面**，实现自动页面跳转、参数同步填入、以及直接触发信号生成或滤波计算，形成无缝的闭环操作。

### 🔄 双交互模式架构
为了满足不同学习阶段和交互场景，系统设计了双交互模式：
1.  **Classic (经典对话模式)**：右侧常规 AI 对话面板展开，主要用于长文本对话、详细的理论公式推导以及全面的系统诊断报告。
2.  **Clar-ball (避让小精灵模式)**：聊天面板收缩，右下角展示具有软质感发光的“呼吸圆球”小精灵。该模式下小球会随鼠标悬停在重要元素上方（如波形图、脑图节点、输入框）并飞跃至目标旁，支持 1.5 秒悬浮语义感知与单击交互，最大限度降低界面遮挡，提供沉浸式实验体验。

## 2. 核心功能及细节描述

AI 助教 Clar 的功能分为四大核心维度：**信号工坊自动生成**、**信号处理工坊自动运行**、**课程图谱与知识演进联动**、以及**高情境感知式对话**。

### 📡 维度一：信号工坊自动生成 (Signal Lab Automation)
在"信号工坊"页面下，用户可以通过自然语言向 Clar 下达生成信号的指令。

#### 1. 支持生成的信号类型
*   **正弦+白噪声 (Sine + White Noise)**：连续正弦波叠加高斯白噪声。
*   **方波+白噪声 (Square + White Noise)**：标准双极性方波叠加高斯白噪声。
*   **三角波+白噪声 (Triangle + White Noise)**：周期三角波叠加高斯白噪声。
*   **双频正弦+白噪声 (Dual-Tone Sine + White Noise)**：两个不同频率正弦波等幅叠加 $x(t) = 0.5[\sin(2\pi f_1 t) + \sin(2\pi f_2 t)] + n(t)$。
*   **窄带高斯随机信号 (Narrowband Noise)**：白噪声通过理想带通滤波器产生的平稳带通高斯过程。
*   **瑞利分布包络信号 (Rayleigh Envelope)**：窄带高斯噪声的正交分量包络 $R(t) = \sqrt{I^2(t) + Q^2(t)}$。
*   **线性调频信号 (LFM - Linear Frequency Modulation)**：扫频信号，频率随时间线性递增。
*   **高斯白噪声 (Gaussian White Noise)**：零均值、单位方差的独立同分布高斯随机过程。
*   **一阶马尔可夫过程 (First-Order Markov Process)**：$x[n] = a \cdot x[n-1] + w[n]$，自回归系数 $a$ 可配。

#### 2. 参数自动解析与控制
Clar 能够从自然语言中准确提取并回填以下控件数值：
*   信号频率 / 中心频率 / 起始频率 ($f_0$，单位：Hz)
*   香农-奈奎斯特采样率 ($f_s$，单位：Hz)
*   信噪比 ($\text{SNR}$，单位：dB)
*   窄带或瑞利分布的信号带宽 ($B$，单位：Hz)
*   线性调频的扫频终止频率 ($f_{end}$，单位：Hz)
*   马尔可夫过程的自回归系数 ($a$，范围 0-0.99)
*   双频正弦的第二频率 ($f_2$，单位：Hz)

#### 3. 模糊/缺省指令智能回填 (重要特性)
若用户只说"*生成一个窄带信号*"或"*来一个正弦波*"而没有提供具体频率和信噪比，Clar **绝对不会**向用户追问参数，而是直接填入以下学术/工程上的**经典默认值**并一键生成：
*   **正弦+白噪声** / **方波+白噪声** / **三角波+白噪声**：频率 = $200\text{ Hz}$，采样率 = $10000\text{ Hz}$，信噪比 = $10\text{ dB}$。
*   **双频正弦+白噪声**：$f_1 = 200\text{ Hz}$，$f_2 = 300\text{ Hz}$，采样率 = $10000\text{ Hz}$，信噪比 = $10\text{ dB}$。
*   **窄带** / **瑞利分布**：中心频率 = $200\text{ Hz}$，采样率 = $10000\text{ Hz}$，信噪比 = $10\text{ dB}$，带宽 = $50\text{ Hz}$。
*   **线性调频 (LFM)**：起始频率 = $100\text{ Hz}$，终止频率 = $400\text{ Hz}$，采样率 = $10000\text{ Hz}$，信噪比 = $10\text{ dB}$。
*   **高斯白噪声**：采样率 = $10000\text{ Hz}$，信噪比 = $10\text{ dB}$。
*   **一阶马尔可夫过程**：$a = 0.9$，采样率 = $10000\text{ Hz}$，信噪比 = $10\text{ dB}$。

#### 4. 多维时频域图表自动切换
Clar 可以根据用户的提问意图自动切换信号工坊内的图表视图：
*   **时域图表**：波形图 (`waveform`)、自相关函数图 (`autocorr`)、互相关函数图 (`crosscorr`)、概率密度函数图 (`pdf`)
*   **频域图表**：幅度谱 (`amplitude`)、功率谱密度 (`psd`)、相位谱 (`phase`)

### ⚙️ 维度二：信号处理扩展工坊自动运行 (Toolbox Automation)
在"工具箱"页面下，Clar 可以根据用户的指令直接配置和执行复杂的信号处理变换和状态估计仿真。

#### 1. 时域：滑动平均系统 (Moving Average Filter)
*   **AI 自动控制**：输入如"*帮我运行 20 点的滑动平均*"时，Clar 会将窗口大小 $N$ 设为 $20$，自动跳转至工具箱页面，并触发 `applyFilter('moving_average')`，实时渲染处理前后的时域波形与对比。
*   **模糊指令处理**：仅说"*运行滑动平均*"，自动默认窗口大小 $N = 10$。

#### 2. 频域：RC 一阶低通滤波器 (RC Low-pass Filter)
*   **AI 自动控制**：输入如"*把截止频率设为 800Hz，运行一阶低通滤波*"，Clar 自动填入截止频率 $f_c = 800\text{ Hz}$，跳转至工具箱，并触发 `applyFilter('rc_lowpass')` 计算冲激响应与输入信号的卷积。
*   **模糊指令处理**：仅说"*跑一下低通滤波*"，自动默认截止频率 $f_c = 500\text{ Hz}$。

#### 3. 频域：维纳最佳线性滤波 (Wiener Filter)
*   **AI 自动控制**：在二阶统计特性（信号和噪声功率谱）已知时，一键触发最小均方误差 (MMSE) 下的最优去噪估值。

#### 4. 卡尔曼滤波状态空间估计仿真 (Kalman Filter Simulation)
针对经典的目标二维状态（位置和速度）跟踪，Clar 可控制运行迭代滤波。
*   **可配参数**：过程噪声强度 $q$、测量噪声标准差 $\sigma_v$ (即代码中的 $r$)、初始状态速度 $v_0$。
*   **AI 自动控制**：用户指示后，系统自动填入参数（如 $q=0.05, r=1.5$），跳转页面并调用 `runKalmanSimulation()`，展示"位置跟踪对比"、"速度估计对比"及"误差协方差 $P_{11}$ 和增益 $K_1, K_2$ 收敛曲线"。
*   **模糊指令处理**：若直接说"*运行卡尔曼仿真*"，参数默认回填：$q=0.1, r=1.0, v_0=1.0$。

### 🕸️ 维度三：课程图谱与知识演进联动 (Knowledge Graph Association)
系统包含精心设计的"课程知识图谱"页面。

*   **可视化图谱交互**：前端使用 ECharts 绘制了一个四层树状/网状的力导向关系图，清晰呈现《随机信号分析》大纲的章节、小节和具体核心知识点。
*   **知识点动态高亮**：当用户向 AI 提问某个知识点时，图谱中对应的节点会实时高亮（放大 + 琥珀色 + 发光阴影），直观展示当前学习上下文。
*   **边缘羽化蒙版**：图谱容器的四周边界采用 CSS `mask-image` X-Y 交叉双向羽化渐变蒙版。节点飘向视口边界时不透明度平滑渐变为 $0$，消除生硬截断。该实现通过 `mask-composite: intersect` 配合 WebKit 专用的 `-webkit-mask-composite: destination-in`，完美兼容 Chrome、Safari 及 ngrok 穿透环境。
*   **点击激活上下文**：在图谱中点击任意知识节点（章节/小节/知识点），前端会自动触发提问发往 Clar，并且在右侧激活"**衍生知识演进面板**"。
*   **双知识卡片推进**：演进面板以具有强烈悬浮质感、渐变边框和深度阴影的玻璃化卡片渲染：
    *   **📌 基石脉络卡**：当前知识点的上游预设理论与推导关联。
    *   **🔥 当前聚焦新知卡**：当前点击知识点的核心数学定义与工程物理含义。
    *   **💡 上轮追索关联推导**：以 KaTeX 完美渲染的高清数学公式及 AI 生成的智能启发式追问按钮。

### 💬 维度四：高情境感知式对话 (Contextual Rich Chat)
Clar 绝非仅有预设问答的普通机器人，她拥有极强的环境自省与交互感知能力。

*   **数字特征实时同步**：当你在信号工坊生成任意参数的信号后，Clar 内部会自动调用 `get_current_signal()`，实时同步该信号实现的均值 $E[X]$、方差 $D[X]$、均方根 $\text{RMS}$、峰峰值和峭度。当你询问"*分析我刚才生成的信号*"时，Clar 会结合这组真实的离散统计特征与连续数学理论，直接在气泡中给出公式推导和误差来源分析。
*   **LTI 系统物理分析**：运行滤波器或卡尔曼仿真后，输出特征同样会同步给 Clar。Clar 能准确告知用户其截止频率处的衰减、自适应滤波下的均方误差收敛状态，并点出因果性物理可实现性的精髓。
*   **全场景屏幕感知 (Universal Screen Sensing)**：在 Clar-ball 模式下，系统通过 `telemetry.js` 监听全局鼠标悬停。当鼠标在图表、输入参数或文字说明上悬停 1.5 秒时，小精灵会自动高亮并飞跃至元素旁，显示磨砂玻璃提示气泡。
    - **图表元数据深度捕获**：系统能精准提取用户当前聚焦的具体图线与子视图类型（例如：时域下的自相关函数图线、频域下的相位谱图线、或雷达追踪的检测漏警分布等）。
    - **脑图 Canvas 节点捕获**：绑定 ECharts 的 `mouseover` 动作，即便在 Canvas 底层绘图块中悬停，也能瞬间解析物理节点的数据与位置。
    - **兜底语义解析**：对于无预设 tooltip 的普通文本或参数标签，通过 DOM 树逐级回溯合成语义文本，确保界面全量可被感知。
*   **主动式物理诊断系统 (Proactive Diagnostics)**：系统在后台静默评估每次仿真的合理性：
    - **Shannon-Nyquist 混叠检测**：若采样率低于最高频率的两倍（$f_s < 2f_{max}$），主动预警并提供混叠公式与抗混叠滤波推导。
    - **滤波器越界异常**：RC 低通滤波的截止频率高于或低于有用信号主频宽时，主动分析信号衰减。
    - **卡尔曼滤波信任博弈**：过程噪声 $q$ 与测量噪声 $r$ 出现极端配比导致估计轨迹发散或过度平滑时，主动讲解 Q/R 博弈。
    - **雷达 CFAR 检测缺陷**：漏警/虚警率高频失控时，主动剖析恒虚警率门限调整。
    - **学习停滞唤醒**：在重要知识点无操作停留过久时，主动发起启发式自测问题。
    - **交互分发**：触发上述情况时，小精灵转为紫色呼吸并飞入 Proactive Card，用户点击后一键切回 Classic 模式展开面板，呈现针对性的 AI 诊断内容。
*   **AI 追问与反问**：针对每一次的讲解，AI 都会结合当前的讨论节点，在气泡底部提炼出 $3$ 个极具启发性和工程实践价值的**智能追问按钮**（如"*为什么在这个阻带下混叠无法完全消除？*"），辅助学生探索。
*   **消息撤回**：发送消息后 5 秒内，用户可以点击"撤回"按钮取消请求。撤回时系统通过 `AbortController` 终止 HTTP 请求、清除 DOM 气泡、并将已发送的文本恢复到输入框。撤回按钮以优雅的收缩动画退场，气泡自然扩展补位。
*   **多行输入**：聊天输入框支持 Enter 发送、Shift+Enter 换行，高度从 56px 自动扩展至最大 120px。

## 3. Clar 搭载的专业知识库内容

Clar 拥有高精度、对标名校课程考核要求的后台实体库：

1.  **概念实体库**：包含"广义平稳 (WSS)"、"维纳-欣钦定理"、"窄带高斯随机信号"、"维纳滤波"、"卡尔曼滤波"等核心节点的理论解构及工程直觉解释。
2.  **公式符号库**：支持行内 $...$ 和独立分行 $$...$$ 的数学表达，如维纳-欣钦变换对、非因果频域维纳传输函数、状态递推与卡尔曼增益方程等。
3.  **避坑与错因库**：整理了学生期末和答辩最容易失分的要点：
    *   *误区一*：误以为广义平稳必然严格平稳（指出唯有联合高斯过程两者才等价）。
    *   *误区二*：认为积分电路后输出仍平稳（纠正不定积分通常会破坏平稳性）。
    *   *误区三*：混淆频域维纳滤波的非因果性与实时在线的霍普夫因果可实现性。
    *   *误区四*：不理解卡尔曼滤波中 $Q/R$ 比值代表的"模型不确定度与测量噪声的信任博弈"。

## 4. 系统的设计小巧思 (Design Craftsmanship & Innovations)

本系统在界面交互、数学可视化、自适应教学和数据拟合方面精心设计了多个"小巧思"，旨在答辩和演示时给老师留下深刻的专业印象：

### 💡 巧思一：零阻碍的"感知-决策-控制"闭环 (Zero-Friction Control Loop)
*   **传统痛点**：大多数 AI 助教只停留在"打嘴仗"阶段，学生询问如何调整参数后，仍需手动在界面拖拽滑块或修改数值，体验割裂。
*   **本校设计**：我们将大模型通过 Function Calling 生成的 JSON 执行命令与前端框架深度融合。当 AI 说"*我已经为您生成了...*"时，底层的输入框参数、下拉框选项已被静默修改，且**立即自动触发网络请求与图表重绘**，页面也已无缝切换到对应的控制台。真正实现了"口令即操作"的现代智能体人机交互。

### 💡 巧思二：防混叠的"交互式奈奎斯特守门员" (Dynamic Shannon-Nyquist Gatekeeper)
*   **传统痛点**：教科书上的采样定理 $f_s \ge 2 f_{max}$ 是死板的公式，学生在仿真时输入错误的采样率容易导致混叠而不知因果。
*   **本校设计**：
    1.  **最大频率自适应测算**：对于线性调频（LFM），最高频率为终止频率 $f_{end}$；对于窄带信号，最高频率为 $f_0 + B/2$。系统能够智能计算当前信号实现下的实际 $f_{max}$。
    2.  **视觉预警机制**：当用户或 AI 设置的采样率 $f_s$ 低于 $2 f_{max}$ 时，采样率输入框背景会立刻变成红色，并在悬浮提示（title）中警告："*采样率已低于临界 Nyquist 频率，将出现频谱混叠！*"；若处于 $2$ 至 $4$ 倍的临界带，则显示橙色警告："*采样率接近下限，重建精度可能不足*"。这极好地充当了互动式教学的教具。

### 💡 巧思三：时域细粒度重建与"离散样本点"对比 (High-Fidelity Waveform Reconstruction)
*   **传统痛点**：在低采样率下（例如 400Hz 信号配合 1000Hz 采样率），如果只用 Plotly 连接采样点，正弦波会缩水成非常难看的折线/三角波，容易误导学生对"连续信号"的认知。
*   **本校设计**：
    *   我们在后端增加了**高分辨率细粒度插值器** (`generate_fine_clean`)。后端会生成一条高达 $2000$ 点的超采样纯净理论曲线 `clean_fine`，与实际采样得到的离散样本点 `noisy` 同步传回。
    *   在前端，系统会同时绘制出平滑的连续蓝色曲线（代表真实的模拟信号 $s(t)$）与红色的离散采样标记点（代表实际采样样本）。时域波形既美观又严谨地展示了"模拟连续信号通过采样离散化"的物理过程。

### 💡 巧思四：边缘羽化流光蒙版 (CSS Mask Aesthetics)
*   **传统痛点**：在力导向知识图谱或长聊天记录滚动时，气泡和节点在边缘会被容器的几何边框直接"切断"消失，视觉感受突兀，缺乏现代网页的细腻度。
*   **本校设计**：
    *   利用 CSS `mask-image` 配合 `linear-gradient` 对聊天容器（`#chat-messages`）、知识衍生面板（`#cards-wrapper`）以及工具箱做了**滚动上下边缘羽化**。当内容滚出视图时会自然渐隐。
    *   特别针对"课程知识图谱"，设计了**X-Y 交叉双向羽化蒙版**。利用标准 `mask-composite: intersect` 结合 WebKit 专用的 `-webkit-mask-composite: destination-in` 遗留标准，使得复杂的拓扑节点和关联边在飘向视口边界时，不透明度平滑渐变为 $0$。该写法完美解决了多重线性渐变在 Safari（包含 ngrok 内网穿透多端访问或 iOS 移动端设备）下可能出现的遮罩失效或渲染不正常的兼容性 Bug。

### 💡 巧思五：多层渐变 border-box 与 3D 浮动感 (Double Gradient Glassmorphism)
*   **传统痛点**：普通的玻璃拟态（Glassmorphism）只使用单一的 `border: 1px solid rgba(...)`，在浅色渐变背景下容易边界不清或缺乏层次。
*   **本校设计**：
    *   我们使用双重 `background` 配置（`padding-box` 控制填充色，`border-box` 配合白到透明的渐变控制边框线），配合 `border: 1px solid transparent`，从而在保留完美圆角的同时，实现了**边框线条本身的透明度渐变**。
    *   配合精密微调的 `transform: translateY(-6px) scale(1.015)` 缓动与双层漫反射投影，当鼠标划过知识卡片和图谱时，卡片呈现出极其柔和的"物理抬升"反馈，为学术严肃的随机信号实验平台赋予了富有生命力的现代拟物交互感。

### 💡 巧思六：流式 JSON 实时解析与零延迟首字 (Streaming JSON Parser)
*   **传统痛点**：AI 以 JSON 格式返回结构化数据时，必须等待整个 JSON 完成后才能解析并展示，首字延迟较高。
*   **本校设计**：
    *   设计了 `JSONReplyStreamParser` 有限状态机。在 AI 流式输出 JSON 的过程中，解析器实时定位到 `"reply"` 字段，逐字符提取其值并推送前端。支持 `\\n`、`\\t`、`\\uXXXX` 等 JSON 转义字符的在线解析。
    *   用户感知的首字延迟趋近于零，对话体验丝滑流畅。

### 💡 巧思七：消息撤回与优雅退场动画 (Message Recall with Graceful Animations)
*   **传统痛点**：用户发送错误消息后无法撤回，需等待 AI 回复完成才能重新提问。
*   **本校设计**：
    *   发送消息后 5 秒内显示半透明玻璃态"撤回"按钮。点击后 `AbortController` 立即终止 HTTP 请求、清除用户和 AI 气泡、并将原文恢复到输入框。
    *   撤回按钮退场动画采用 `max-width: 0` + `padding: 0` + `margin: 0` + `scale(0.7)` + `opacity: 0` 五维同步收缩过渡。由于 `.chat-msg.user` 设置了 `gap: 0`，按钮消失时不会产生任何布局空隙跳动，气泡自然平滑地扩展到满宽。

### 💡 巧思八：几何科技背景与等距网格 (Geometric Tech Background)
*   **传统痛点**：纯色或简单渐变背景缺乏科技感和视觉层次。
*   **本校设计**：
    *   背景采用三层叠加设计：slate-blue 渐变底色 + SVG 等距三角网格 (`body::before`) + 四个漂浮旋转的 SVG 几何图形（两个六边形 + 两个三角形）。
    *   几何图形使用 `linearGradient` 填充 + 半透明描边，配合 30-45 秒的长周期 `float-rotate` 动画，营造出优雅而不喧宾夺主的科技感氛围。
